Definetely the best template engine out there, and I am not saying that because I wrote it, but because it is true. Fast, asynchronys or synchronues. Templates, templates with body ( layouts ), templates with several bodies. Shows where error occured if there are errors in the view Scoped. Comments. Nested comments. Inline code. Give it a try! You won't regret it! Be careful in asynchronues mode, threads will go ahead and perform tasks simualtenuesly, so if you change a value somewhere in a subtemplate, it might not be ready further down in your parent template when that is invoked. You shouldn't however relying on changing the model used in a parent template in a subtemplate. Bad design! Hook into the current rendering if you need to perform an ajax requrest and to be part of your response.
